Node.js & npm

What is it and why is it useful

According to its website, Node.js® is an asynchronous event-driven JavaScript runtime. This definition, while meaningful, is a little jargon-y.

In this context, asynchronous means that multiple processes can be executed at the same time. This can really improve the speed with which a Node.js server is able to process users’ requests, since there’s no bottlenecking of requests that must be processed one after the other.

Event-driven just means that the Node.js server waits for a user to do or input something before it reacts in some way.

A runtime environment is often described as the execution environment provided to an application by the operating system- it is used by software developers to test and debug their applications outside of the browser.

And finally, the Node.js website specifies Javascript as the programming language because that’s just the language that it is build for. We’re not talking about Ruby here!

Node.js is commonly used when building APIs and is particularly useful for data-heavy apps that may need to scale up in the future (ie accommodate more users and requests).

Npm, or the Node Package Manager, is a package manager for Node.js. In fact, one of the great benefits of using Node.js is the huge quantity of code packages available through npm. Npm also includes a command line interface (CLI) which can be used to download and install those code packages.
